# communication
* すべての説明・コメント・コミットメッセージは基本的に日本語で書くこと。
* コード内の識別子（変数名・関数名）は英語を使うこと。
* 曖昧な指示に対しては明確化を求めること。
* 推測する場合は根拠を明記し、事実と推測を混同しないこと。


# coding rules
* 主要エントリー関数は、可能な限り関数などを用いてシンプルにし、理解しやすい直列フローで書くこと。
* すべてのタスク管理は ルート直下の `TODO.md` に一本化し、その内容に従うこと。
* 新規タスクが発生した場合、`TODO.md`に追記してから実行すること。
* タスクを完了した場合、該当項目に完了マーク`[x]`を付与する差分案を提示すること。
* `TODO.md`と実装内容に不整合がある場合、ユーザーに確認と同期を提案すること。
